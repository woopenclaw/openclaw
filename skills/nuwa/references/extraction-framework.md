# 思维框架提炼方法论

## 一、心智模型识别的三重验证

一个论点要被认定为「心智模型」而非「随口一说」，必须通过三重验证：

### 验证1: 跨域复现
同一思维框架出现在此人讨论的至少 2 个不同领域。

例：纳瓦尔的「杠杆」——
- 财富创造中谈杠杆（代码、媒体、资本、劳动力）
- 个人成长中谈杠杆（特定知识 + 杠杆 = 复利）
- 职业选择中谈杠杆（选有杠杆效应的工作）
→ 跨 3 领域复现 → 真正的心智模型

### 验证2: 有生成力
用这个模型可以推断此人对新问题的可能立场。

例：芒格的「逆向思维」——
- 面对「如何成功」→ 先想「如何确保失败」
- 面对「如何投资」→ 先想「如何亏光所有钱」
→ 能生成新推断 → 真正的心智模型

### 验证3: 有排他性
不是所有聪明人都这样想，体现此人独特视角。

例：「反脆弱」是塔勒布的，不是通用智慧
→ 有区分度 → 值得提炼

**降级规则**：
- 3 重通过 → 心智模型
- 1-2 重通过 → 降级为决策启发式
- 0 重通过 → 丢弃

## 二、六路调研维度

| Agent | 维度 | 采集目标 | 关键搜索词模板 |
|-------|------|---------|--------------|
| 1 | 著作/长文 | 书籍、长篇文章、博客 | "[人名] book/著作", "[人名] essay/文章" |
| 2 | 播客/访谈 | 播客出演、视频访谈、演讲 | "[人名] interview/访谈", "[人名] podcast", "[人名] speech/演讲" |
| 3 | 社交媒体 | Twitter/X、微博、知乎、公众号 | "[人名] twitter/X", "site:zhihu.com [人名]" |
| 4 | 批评者视角 | 对此人的批评、反驳、争议 | "[人名] criticism/批评", "[人名] controversy/争议", "[人名] wrong" |
| 5 | 关键决策 | 重大决策及其推理过程 | "[人名] decision/决策", "[人名] why [关键事件]" |
| 6 | 人生时间线 | 重要经历、转折点 | "[人名] biography/传记", "[人名] career/经历", "[人名] timeline" |

**每个 Agent 的产出标准**：
- 至少 5 个高质量来源
- 每条信息标注来源 URL
- 原文引用用 `>` 格式
- 标注信息质量：一手（本人产出） vs 二手（他人分析）

## 三、表达DNA分析维度

| 维度 | 提取内容 |
|------|---------|
| 句式偏好 | 长句/短句、疑问/陈述、类比密度 |
| 词汇特征 | 高频词、专属术语、禁忌词 |
| 节奏感 | 先结论还是先铺垫、转折方式 |
| 幽默方式 | 讽刺/自嘲/荒诞/冷幽默/不幽默 |
| 确定性表达 | 「我不确定」型 vs 「很明显」型 |
| 引用习惯 | 爱引谁、引什么类型 |

## 四、信息不足时的处理

| 情况 | 处理 |
|------|------|
| 某维度公开信息很少 | 标注「信息不足，此维度为推测」 |
| 只有二手信息 | 降低置信度，标注为「据[来源]报道」 |
| 信息相互矛盾 | 并列呈现，让用户判断 |
| 此人刻意不公开某方面 | 注明「此人对这个话题保持沉默」 |

## 五、人物Skill vs 主题Skill

| 维度 | 人物Skill | 主题Skill |
|------|----------|----------|
| 核心 | 一个人的思维方式 | 一个领域的思维工具箱 |
| 心智模型来源 | 主要来自一个人 | 综合多人视角 |
| 表达风格 | 模拟此人表达 | 中性但专业 |
| 矛盾处理 | 保留个人内在矛盾 | 呈现流派间分歧 |
| 验证方式 | 对比此人已知立场 | 对比领域内共识 |

## 六、质量自检清单

### 心智模型
- [ ] 每个模型都有至少 2 个不同领域的证据？
- [ ] 模型数量在 3-7 个之间？
- [ ] 每个模型都有明确的应用场景和局限？
- [ ] 模型之间有张力但不矛盾？

### 表达DNA
- [ ] 读起来有辨识度，不像通用 AI？
- [ ] 没有过度模仿变成漫画化？
- [ ] 抓住了核心特征而非表面模仿？

### 决策启发式
- [ ] 每条规则都有具体案例支撑？
- [ ] 可以被新情况触发？

### 诚实边界
- [ ] 明确写了做不到什么？
- [ ] 标注了信息源和调研时间？
- [ ] 承认了信息不足的维度？

### 整体
- [ ] 用此人的眼睛看一个新问题，能得到有价值的视角？
- [ ] 不是原话拼凑，而是框架的运行？
- [ ] 删掉名字后，还能认出这是谁的思维方式？
